# Cell Skill

Cell Skill 是面向 Codex 的科研任务总入口。它将复杂研究请求拆分为最小且有序的工作流，调用合适的原生工具或已安装技能，并对证据、方法、可重复性和交付质量进行统一检查。

它适合文献检索、实验设计、数据分析、科研绘图、组学、生物信息学、蛋白质与结构、化学与临床证据、论文写作、审稿、演示文稿、专利和跨阶段科研任务。

## 核心原则

Cell Skill 始终仅使用 Codex 原生工作模式和 Codex 工具工作流。

不得使用、调用、模仿、路由或回退到任何 GPT work 模式、GPTWORK 工作流、基于 GPT 的工作流抽象、提示约定或任务处理框架。目录名称、项目名称、用户界面标签或第三方适配器均不得覆盖此要求。

## 主要能力

- **证据研究**：文献发现、全文获取、引用核验、证据综合和论断审计。
- **数据分析**：数据剖析、清洗、统计、建模、组学分析和可重复计算。
- **生物学研究**：基因、变异、序列、蛋白质、结构、通路、表达和调控证据。
- **化学与医学**：化合物、靶点、药理、安全性、临床试验和临床证据。
- **科研表达**：论文图、稿件、审稿意见、返修信、基金申请、PPT、专利和数据声明。
- **质量控制**：区分观察、计算、解释和推测，检查来源、统计、标识符、单位及可重复性。

## 工作流程

Cell Skill 按八个阶段执行科研任务：

1. **Frame**：明确科学问题、研究对象、受众、交付物和判断标准。
2. **Inspect**：检查输入数据、文件、来源、版本和完整性。
3. **Design**：选择方法，定义比较、对照、协变量、终点和统计策略。
4. **Acquire**：从权威来源获取必要证据与数据，并记录来源和版本。
5. **Analyze**：使用成熟工具分析，检查覆盖度、收敛、样本损失和异常结果。
6. **Interpret**：保持结论与证据对应，区分相关性、因果性和推测。
7. **Communicate**：根据受众生成图表、论文、报告、演示或其他成果。
8. **Verify**：应用相关质量门槛，修正问题并报告未完成的验证。

## 安装

### 使用 Codex 技能安装器

向 Codex 提交：

```text
安装技能 https://github.com/YongliZhu998/cell-skill/tree/main
```

### 手动安装

将仓库克隆到 Codex 技能目录：

```powershell
git clone https://github.com/YongliZhu998/cell-skill.git "$HOME\.codex\skills\cell-skill"
```

重新打开 Codex，使技能被重新发现。

## 使用

可以直接调用：

```text
$cell-skill 检索某个基因的功能证据，并整理成可追溯的研究报告
```

```text
$cell-skill 分析这份实验数据，选择合适的统计方法，制作论文图并解释结果
```

```text
$cell-skill 根据论文、实验记录和结果图，审查论证链并提出补强方案
```

```text
$cell-skill 查询某个药物靶点的结构、生物活性和临床试验证据
```

当任务跨越多个阶段时，Cell Skill 会按依赖关系组织流程。例如：

```text
实体与标识符解析 -> 数据库查询 -> 证据综合 -> 数据分析 -> 科研绘图 -> 论文写作 -> 质量检查
```

## 可选适配器

Cell Skill 可以调用当前环境中已安装的专业技能，但不依赖某个固定技能包。适配器只负责边界清晰的阶段，最终科学结论和质量控制仍由 Cell Skill 统一负责。

如果没有合适的适配器，Cell Skill 会使用 Codex 原生工具和成熟科学库直接完成任务。

## 质量与可重复性

Cell Skill 要求：

- 优先使用原始研究、官方数据库、文档化 API 和成熟分析库。
- 保留物种、基因组版本、数据库标识符、单位、样本定义和统计估计目标。
- 不伪造引文、实验结果、数据库记录、方法或工具输出。
- 保持原始输入不变，使数据转换和分析步骤可追踪。
- 对随机过程设置种子，并优先提供可重新生成结果的脚本或工作流。
- 明确标记缺失证据、未解决的不确定性和无法执行的检查。
